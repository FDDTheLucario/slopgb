# SPC700 — SNES APU CPU (SGB audio, Phase 2)

Self-contained, cycle-accurate SPC700 (Sony S-SMP) core in
`crates/slopgb-core/src/sgb/spc700/`. Runs the Super Game Boy's SNES-side sound
driver. **Standalone**: no dependency on the GB CPU/PPU/BIOS. Dead code w.r.t.
the GB path until Phase 3 wires the S-DSP + SGB command stream, so it is
golden-safe by construction (the only shared-file touch is the one-line
`pub mod sgb;` in `lib.rs`).

This is **Phase 2 of 3** for SGB audio: Phase 1 = SGB commands/border (separate),
Phase 3 = S-DSP + wiring (plugs into the [DSP seam](#dsp-seam-phase-3)).

## Register / flag model

`A`, `X`, `Y`, `SP` (8-bit), `PC` (16-bit). `A`/`Y` form the 16-bit pair `YA`
(`Y` = high byte). `PSW` = `N V P B H I Z C` (stored unpacked as [`Psw`], packed
to/from a byte only for `PUSH/POP PSW`, `BRK`, `RETI`):

| Bit | 7 | 6 | 5 | 4 | 3 | 2 | 1 | 0 |
|-----|---|---|---|---|---|---|---|---|
| Flag| N | V | P | B | H | I | Z | C |

`P` (bit 5) is the **direct-page select**: `0` → `$00xx`, `1` → `$01xx`. All
`dp`/`dp+X`/`dp+Y`/`(X)`/`(Y)` addressing honours it; `dp+X`/`dp+Y` and the
indirect pointer bytes wrap **within** the 256-byte page. `!abs+X`/`!abs+Y` index
across the full space.

Reset (`Spc700::reset`, bsnes `SMP::power`): IPL ROM enabled, `PC` = word at the
reset vector `$FFFE/$FFFF` (`$FFC0` while the IPL is mapped), `A=X=Y=SP=0`,
`PSW=0`. The IPL immediately sets `SP`/`X`, so their reset values are inert.

## Memory map

`ram.rs`. 64 KB RAM underlies the whole space.

| Range | Contents |
|-------|----------|
| `$0000-$00EF` | RAM (direct page 0) |
| `$00F0-$00FF` | I/O registers (`ports.rs`) |
| `$0100-$01FF` | RAM (page 1, stack; direct page 1 when `P=1`) |
| `$0200-$FFBF` | RAM |
| `$FFC0-$FFFF` | 64-byte IPL boot ROM when `$F1` bit 7 set, else RAM |

The IPL ROM (`ram::IPL_ROM`) is the invariant 64-byte SNES APU boot loader,
verified against fullsnes ("IPL Boot ROM") and bsnes `SMP::iplrom`. Reads in
`$FFC0-$FFFF` return the ROM while mapped; **writes always fall through to RAM**.

## I/O registers (`$F0-$FF`)

`ports.rs`.

| Reg | Read | Write |
|-----|------|-------|
| `$F0` TEST | stored value | stored |
| `$F1` CONTROL | stored (bits 0-2,7) | timer enables (0-2), port-clear strobes (4,5), IPL enable (7) |
| `$F2` DSPADDR | value | DSP register select |
| `$F3` DSPDATA | DSP read | DSP write |
| `$F4-$F7` PORT0-3 | input latch (SNES→APU) | output latch (APU→SNES) |
| `$F8/$F9` AUX | stored | stored |
| `$FA-$FC` T0-2 target | 0 (write-only) | timer divisor (0 = 256) |
| `$FD-$FF` T0-2 out | 4-bit counter, **read clears** | ignored |

### Comm-port latch model

Each `$F4-$F7` port is **two** independent latches: `port_in` (SNES writes via
`snes_write_port`, APU reads) and `port_out` (APU writes, SNES reads via
`snes_read_port`). An APU write to `$F4` is invisible to a later APU read of
`$F4`; the two sides talk only through the paired opposite latch. `$F1` bit 4
clears input ports 0/1, bit 5 clears 2/3 (one-shot strobes; read back as 0).

### Timers

`timers.rs`. T0/T1 at 8 kHz (1.024 MHz ÷ 128), T2 at 64 kHz (÷ 16). An 8-bit
stage counts up to its target (0 → 256); on match it resets and bumps the 4-bit
output. Reading the output clears it. A `$F1` enable 0→1 edge resets stage +
output. Driven at whole-instruction granularity from `step()`'s cycle count
(ample for a polling driver; upgrade to per-cycle only if a test needs
sub-instruction timer phase).

## DSP seam (Phase 3)

`ports.rs`. `$F2` selects a DSP register, `$F3` reads/writes it. Phase 3 attaches
the S-DSP with `Spc700::attach_dsp(Box<dyn Dsp>)`:

```rust
pub trait Dsp {
    fn read(&mut self, addr: u8) -> u8;      // raw $F2 forwarded (incl. mirror bit)
    fn write(&mut self, addr: u8, val: u8);  // raw $F2 forwarded
    fn tick(&mut self, cycles: u32) {}       // clocked from each step()'s cycles
}
```

The raw `$F2` address is passed through, so the DSP sees the read-only mirror bit
(`$80-$FF`). With no DSP attached, a 128-byte shadow answers (read mirrors low 7
bits; writes with bit 7 set ignored) so the SPC700 is self-consistent standalone.
`step()` ticks the attached DSP by the instruction's cycle count. Phase 3 also
uses `snes_write_port`/`snes_read_port` for the SNES↔APU comm path.

## Cycle timing

`step() -> u32` executes one instruction and returns its cycle count; timers +
DSP are advanced from that. Base counts are in `mod.rs::CYCLES` (fullsnes "SPC700
Opcodes"), **cross-checked exhaustively** against the `SingleStepTests/spc700`
cycle traces. Conditional ops (the 8 `Bcc`, `BBS`/`BBC`, `CBNE`, `DBNZ`) store
the not-taken count and add +2 when taken. The SPC700 runs at 1.024 MHz.

`SLEEP` (`EF`) / `STOP` (`FF`) halt the oscillator (`stopped` flag, cleared only
by reset). The hardware trace snapshots a 7-cycle fetch-then-spin, so they report
7 cycles; a halted `step()` returns 2 and keeps ticking the timers/DSP (they run
on real hardware while the CPU is halted).

## Opcode coverage

**All 256** opcodes implemented, split by category: `ops_alu.rs`
(OR/AND/EOR/CMP/ADC/SBC, shifts/rotates, INC/DEC, ADDW/SUBW/CMPW/INCW/DECW,
MUL/DIV, DAA/DAS, XCN), `ops_mov.rs` (all MOV loads/stores incl. `(X)+`
auto-increment + addressing-mode helpers), `ops_branch.rs` (Bcc/BRA/JMP/CALL/
RET/RETI/BRK/TCALL/PCALL/CBNE/DBNZ), `ops_bit.rs` (SET1/CLR1/BBS/BBC/TSET1/TCLR1
+ AND1/OR1/EOR1/MOV1/NOT1 on 13-bit `addr = word & 0x1FFF`, `bit = word >> 13`),
`ops_misc.rs` (NOP, flag/control ops, SLEEP/STOP). Cited quirks:

- **`DIV YA,X`** — the S-SMP overflow quirk (bsnes `instructionDivide`): `V` set
  when the quotient won't fit in 8 bits, `H = (Y&15) >= (X&15)`, N/Z from the
  final `A` even on overflow, and the non-restoring-divider wrap in the overflow
  case. `X == 0` routes to the `else` branch (divisor `256 - X`), never a real /0.
- **`DAA`/`DAS`** — bsnes decimal-adjust: high-nibble `+/-0x60` on `C || A>0x99`,
  low-nibble `+/-0x06` on `H || (A&15)>9` (inverted for `DAS`).
- **`ADDW`/`SUBW`** — two 8-bit adds so `H`/`V` come from the **high** byte
  (H = carry out of bit 11), C = carry out of bit 15, N = bit 15, Z = full word.
- **`CMP`/`CMPW`** — set only N Z C (no V, no H, unlike SBC/SUBW).
- **`CLRV`** — clears **both** V and H.
- **`TSET1`/`TCLR1`** — N,Z from `A - mem_original` (a comparison), not the
  written value.
- **`MUL YA`** — N,Z from `Y` (the result's high byte).
- **`CBNE`/`DBNZ`** — affect **no** flags.

## Tested — how

`spc700_tests/`, compiled as `super::tests`:

- **`harness.rs::singlestep_conformance`** (`#[ignore]`d) — the real oracle:
  the `SingleStepTests/spc700` v1 suite (MIT), **1000 hardware-traced cases per
  opcode = 256,000 total**, full pre/post register + RAM state + cycle count.
  **256000/256000 pass.** Run with the suite unpacked:
  ```sh
  SPC700_TESTS_DIR=/path/to/spc700/v1 \
    cargo test -p slopgb-core --lib sgb::spc700 -- --ignored singlestep
  ```
  (The `v1/XX.json` files are ~81 MB; fetched from
  `github.com/SingleStepTests/spc700`, not vendored in-tree.) A tiny std-only
  JSON parser reads them (no new deps).
- **`harness.rs::opcode_cycle_table_smoke`** (always runs) — executes all 256
  opcodes, asserts each consumes its documented base count (or base+2 for
  conditionals). Guards the cycle table + proves the dispatch never panics.
- **`ops.rs`** — per-opcode / per-flag unit tests emphasising the quirks above
  (DIV overflow + half-carry + div-by-zero, MUL, DAA/DAS boundaries, ADDW/SUBW
  half-carry, MOV1/NOT1 13-bit addressing, TSET1/TCLR1, operand order of
  `dp,dp`/`dp,#imm`, `P`-flag paging, page-wrap pointers, CBNE/DBNZ flag-inertia).
- **`timers.rs`** — 8/64 kHz dividers, read-and-clear, target-0-means-256,
  4-bit wrap, disabled freeze, enable-edge reset, three-timer independence.
- **`ipl.rs`** — reset enters `$FFC0`, IPL overlay toggle, the `$AA`/`$BB`
  handshake (boots the real IPL), comm-port latch separation, control strobes,
  the DSP seam (shadow self-consistency + forwarding to an attached mock + tick).

## Unverified / left for Phase 3

- **S-DSP** — not implemented (the seam is here; Phase 3 fills it). The shadow is
  a plain register file, not a real DSP.
- **Sub-instruction timer/DSP phase** — timers driven at whole-instruction
  granularity. Fine for a polling driver; revisit only if a Phase-3 test needs it.
- **`$F0` TEST semantics** — stored/echoed. The real register gates undocumented
  hardware test modes the SGB driver never touches.
- **`$F0`/`$F1` read open-bus** — returned as the stored value (self-consistent);
  real hardware reads are open-bus. The driver never reads them.
- **SNES↔APU wiring + boot** — Phase 3 (loading the SGB sound program over the
  ports via the IPL handshake, then running the driver).
