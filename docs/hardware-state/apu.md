# APU

- Post-boot APU is warmed ~1 emulated second so the boot beep's envelope is decayed at hand-off (PCM12/FF76 reads $00, NR52 keeps ch1 status) — don't "simplify" the warmup away.
- APU follows SameBoy's countdown model (`src/apu/`): pulse/noise step on a machine 2 MHz grid (`Apu::phase`, bit 1 = `lf_div`), triggers anchor to it, the duty bit/LFSR sample is LATCHED at expiries, NR52 power-on resets the divider chain (+DIV-event skip glitch when the DIV-APU bit is high), envelopes use the countdown + rising-edge-arming + lock scheme, noise runs a free-running 14-bit counter that triggers do NOT reset. Same-suite apu is green except freq_change_timing revision variants and ch4 align/freq_change (NR43 corruption tables: upstream-documented non-deterministic) — see the baseline comments before touching.
- Ch1 sweep is SameBoy's calculation-countdown machinery (`pulse.rs`): the 128 Hz fire writes the frequency at once, but the shadow/addend refresh + overflow check complete only `reload_timer + shift` 1 MHz cycles later (kills trail fires/triggers by several M-cycles), NR10 writes hit the live machinery (zombie step, cleared-shift pause, completed-addend negate-clear kill — the E form of the old-negate bit per the §CGB-revision-policy companion rule, NOT SameBoy's ≤C forced-true), and triggers hold shadow refreshes for `channel_1_restart_hold` 2 MHz cycles. Same-suite channel_1_sweep + restart + restart_2 (the README's "even SameBoy-E fails it" ROM) all pass. Residual gambatte ch1_init_reset_sweep_counter_timing rows need the 128 Hz grid phase pinned <4 dots against the instruction stream per model — see the baseline comment; whole-M-cycle ordering tweaks break same-suite.
