# OAM DMA & VRAM DMA

- OAM DMA bus conflicts follow gambatte-core memory.cpp exactly (`interconnect.rs` `DmaSrcKind` + page masks): conflicted writes derail into the in-flight OAM slot (DMG WRAM src wire-ANDs, via `dma_pending_oam` — each copied byte commits to OAM at its cycle's *end*), CGB redirects WRAM-region accesses to the FF46-bit-4 page, CGB sources ≥ $E0 read $FF, FF46 rewrites retarget mid-flight, and CGB-C keeps 24 B of mirrored extra RAM at FEA0-FEFF (AGB: nibble echo). gambatte `oamdma/` is the oracle; residual clusters are commented in the baseline.
- OAM DMA × VRAM DMA composition (`Interconnect::oam_dma_bus_capture`): while a VRAM DMA owns the bus the OAM DMA makes no source reads — it advances once per stolen M-cycle (on the cycle's *last* byte; every byte in double speed) latching the stolen bus byte into OAM[hdma_src & $FF] (≥ $A0 → CGB-C extra OAM RAM), skipping its own copies for those positions; frozen entirely while the core clock is gated (gambatte memory.cpp `dma()`). A *halt-mode* wake additionally runs one OAM-DMA catch-up M-cycle before the CPU's first post-wake cycle (SameBoy `GB_cpu_run` halt exit `dma_cycles=4; GB_dma_run`); the speed-switch pause exit deliberately does NOT (gambatte `oamdmasrcC0_speedchange_readC000` pins the un-caught-up resume).
- CGB VRAM DMA is a gambatte-shaped request engine (`Interconnect::vram_dma_req`): FF55 is the live register (cancel latches the *written* length | $80); the dot-exact mode-0 entry — led by one dot, `Ppu::hdma_trigger_level`, gambatte xpos `lcd_hres+7` — flags one block; requests steal the bus at the head of the CPU's next bus op, with reads yielding to a same-cycle trigger while in-flight writes commit first (hdma_late_destl vs hdma_start `_1`/`_2` pairs); each service ends with one teardown M-cycle; the 16-bit dest counter terminates at the 0x10000 crossing (no VRAM wrap — gambatte `dma_dst_wrap_2` overrides the old SameBoy-derived wrap); enabling with the LCD off copies one block immediately; an LCD disable kills an armed transfer but leaves FF55 reading active. Residual `_2`/`a-phase` parity rows are documented swaps in `baselines/gambatte.txt` — they need sub-M-cycle phase, don't chase them with whole-dot timing.
