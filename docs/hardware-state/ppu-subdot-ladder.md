# PPU — sub-dot event-phase ladder (half-dot / eighth grid)

Lifts the class-H/A sub-dot floor incrementally: keep the pixel pipe dot-clocked, add sub-dot resolution only on the EVENT layer (observers sample at their own phase, events commit at their own phase). gambatte hwtests evaluate at a cc+2 access offset; our tick-then-access is cc+4 (M-cycle end); gbmicrotest (also hardware) pins cc+4 — the model must reproduce both. Full per-increment history (incl. dead-ends) is in memory `slopgb-subdot-eventphase-ladder.md`.

The 9-piece flip decomposes into two RESOLUTION TIERS; Tier 2 is the structural root and the make-or-break.** Six-agent seam-map workflow (`wf_dd561556`) + first-hand T-by-T trace of SameBoy `sm83_cpu.c` dispatch confirmed the organizing fact:
  - **Tier 1 (whole-M-cycle leading-edge reads) is DONE + sufficient** for FF41 + OAM/VRAM/palette accessibility + the kernel pair. Mechanism: `leading_edge_sample` (`interconnect/cycle.rs:16`) reads the PPU *before* `tick_machine` (`interconnect.rs:706`) → cc+0. The discarded `clock.read()`/`clock.write()` returns are pure bookkeeping; consuming them changes nothing while `tick_machine` advances the PPU in flat 4-dot quanta.
