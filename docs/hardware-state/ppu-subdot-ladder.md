# PPU — sub-dot event-phase ladder (half-dot / eighth grid)

Lifts the class-H/A sub-dot floor incrementally: keep the pixel pipe dot-clocked, add sub-dot resolution only on the EVENT layer (observers sample at their own phase, events commit at their own phase). gambatte hwtests evaluate at a cc+2 access offset; our tick-then-access is cc+4 (M-cycle end); gbmicrotest (also hardware) pins cc+4 — the model must reproduce both. Full per-increment history (incl. dead-ends) is in memory `slopgb-subdot-eventphase-ladder.md`.

- **PHASE B (the cc-exact reclock) — ATOMIC-CORE ARCHITECTURE FIXED + thesis protocol (2026-06-23, mandated). The 9-piece flip decomposes into two RESOLUTION TIERS; Tier 2 is the structural root and the make-or-break.** Six-agent seam-map workflow (`wf_dd561556`) + first-hand T-by-T trace of SameBoy `sm83_cpu.c` dispatch confirmed the organizing fact:
  - **Tier 1 (whole-M-cycle leading-edge reads) is DONE + sufficient** for FF41 + OAM/VRAM/palette accessibility + the kernel pair. Mechanism: `leading_edge_sample` (`interconnect/cycle.rs:16`) reads the PPU *before* `tick_machine` (`interconnect.rs:706`) → cc+0. The discarded `clock.read()`/`clock.write()` returns are pure bookkeeping; consuming them changes nothing while `tick_machine` advances the PPU in flat 4-dot quanta.
  - **Tier 2 (sub-M-cycle dispatch reclock) is the BLOCKER and the thesis.** The interrupt-dispatch retime is **−2 T = half an M-cycle** (`dispatch_vector_retime`, `cycle_clock.rs:149`: `pending-=2; flush; pending=2`). Traced SameBoy: after the low push the IF-ack/vector latches at M5+2 dots, and the **`pending=2` re-park shifts the vector fetch AND the first handler instruction's reads 2 dots earlier — until the instruction-boundary flush** (`step` → `bus.flush_pending`, `cpu/execute.rs:21`). That is the "re-frames every read" mechanism. It REQUIRES the PPU (and timer/APU/serial) advance to be slaved to the clock's `pending` (deferred-commit), which is **B1** (split `tick_machine` `for cc in 1..=4`, `interconnect/tick.rs:46`, into pre/post-commit spans driven by the clock; needs `timer/apu/serial` `tick(dots)` variants, byte-identical at full-M-cycle). Switching the PPU advance eager(+4/op)→deferred(advance-prev-park) shifts the absolute frame one M-cycle → invalidates every boundary calibration (`vis_early`/`mode3_entry_dot`/A8 dot-254 dispatch) → **B1+B2+B3+recalibration must land together as the atomic core (flag-off byte-identical trivially since flag-gated; flag-on is red until convergence).**
  - **Why this is NOT yet refuted:** the 2026-06-21 "DISPATCH-RECLOCK REFUTED" measured the PPU-side dispatch move (B3, 254→252) *alone* → m0int 0→3, intr_2_0 PASS→FAIL. The thesis claims the CPU-side retime (B2, the `pending=2` reframe) is the missing half. **B2 has never been wired** (`dispatch_vector_retime` is dead outside `cycle_clock_tests`). So B2+B3 together is the untested make-or-break.
