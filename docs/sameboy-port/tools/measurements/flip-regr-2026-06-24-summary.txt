# ===== PER-FAMILY DIAGNOSIS (2026-06-24, dual-emulator traced) =====
# REFUTATION #1 (read frame): literal "read +4" is unimplementable.
#   - uniform whole-dot +4 (read AND boundary) = NO-OP (preserves all orderings).
#   - "sample 4 later" = cc+4 = production tick-then-access → FAILS kernel m2int
#     (probe OFF: m2int want3 got0). cc+0 leading-edge read is essential.
#   - real read-frame residual = ~2-dot SUB-M-cycle read-vs-boundary phase
#     (slopgb m0int dispatch254/read256 gap2; SameBoy 257/261 gap4). Eighth-grid
#     (event_phase/lead_eighths, S7); cc-collapse-limited.
#
# RECLASSIFICATION #2 (window, 107 — NOT the read-wall): per-config mode-3 LENGTH
#   geometry, opposite-direction errors. Traced DMG:
#     m2int_wx00_m3stat_2 want0: slopgb dot260 mode3 / SameBoy cfl265 mode0
#       (slopgb mode-3 too LONG).
#     late_wy_10to0_ly1_1 want3: slopgb dot260 mode0 / SameBoy cfl260 mode3
#       (read dots MATCH 260=260 → PURE length bug: slopgb mode-3 too SHORT).
#   Fix = tier2-gated window mode-3 length port (proj/lead per wx/scx/late-WY
#   trigger) vs SameBoy; proj/lead is production-shared + kernel-pinned (bare exit)
#   → must be a tier2 parallel calc. NOT early_lead (it's a uniform per-arm shift;
#   can't fix opposite-direction length). NOT cc-collapse.
#
# m1 + lycEnable (52): multi-faceted mode-1/LYC/IF-delivery.
#   want=E0 got=E2 (19) spurious STAT IRQ + want=E2 got=E0 (6) missed = the
#   rising-edge-engine IF DELIVERY class (tick.rs if_late/level re-arm).
#   want=3 got=1 (19) / want=1 got=3 (15) = mode-1↔mode-3 bit reads (vblank/wrap
#   entry under the deferred frame). No single lever.
#
# sprites (87): mode-3 sprite geometry (L2 class, render). halt (32): cc-collapse
#   sub-M-cycle wake wall (S7, proven). speedchange/dma 13 (DS, S6/S7).
#
# VERDICT: no clean one-slice win remains; flip = multi-session port across
#   window-geometry + sprite-geometry + m1/lyc-delivery + the S7 sub-dot wall.
#   Each needs SameBoy per-config ground truth + a tier2-gated parallel calc.
