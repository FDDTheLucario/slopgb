# bgb-clone TDD plan

Generated by `/tdd-test-plan`. Functional-1:1 clone of bgb's debugger + VRAM viewer +
I/O map on slopgb's own software renderer (winit + softbuffer, no GUI deps; core std-only,
no unsafe). Spec + real screenshots: [`docs/bgb-reference/README.md`](bgb-reference/README.md).

Sequence: **Layer A** (core debug API, no UI) → **Layer B** (software-UI toolkit) →
**Layer C** (the windows, one at a time). Each `<task>` is one red-green-refactor cycle.
Tasks tagged `[vdiff]` also need real-bgb visual diffing (run bgb on this machine,
screenshot, compare) on top of their unit assertions.

A0 → A1 → A2/A15 → (Layer A done) → B1/B2 → B3 → B-widgets → B12 →

then **C-Debugger**, **C-VRAM**, **C-IO**, **C-integration** — each its own review+commit.
