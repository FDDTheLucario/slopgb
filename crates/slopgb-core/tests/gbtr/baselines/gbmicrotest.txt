# gbmicrotest known-failure baseline (ratchet) — one case key per line.
# Discovered empirically on this emulator (Phase B); shrink as fixes land,
# never grow.
#
# Floor classes (letters per the index in baselines/gambatte.txt):
# dma_basic and oam_int_if_level_d are class F (asset defect / real
# hardware divergence — never fix in core); the win*_b + ppu_sprite0_*_b
# rows are class H one-dot conflicts (lift: sub-dot IF-flop modeling —
# their gambatte window/late_* siblings pin the opposite dot, so don't
# chase one-sidedly); the remaining oam_int/int_oam/lcdon/line_144/
# stat_write_glitch rows are line-0/lcd-enable/line-144 OAM-pulse chains
# not yet derived, and halt_op_dupe_delay is halt-bug timing, untriaged
# (both open work, not floor).
#
# Re-baselined 2026-06 twice: the DMG post-boot LCD phase pin (70164,
# the poweron tables) + STAT mode-2 pulse wave flipped 60 rows, and the
# mode-0 event-grid re-derivation (flip/IRQ at pipe end - 2; see
# ppu/render.rs m0_flip_events) flipped 46 more (the hblank_int_scx*
# dispatch/_if/_nops grids, int_hblank_* incl. all 8 halt rows,
# hblank_int_l0, ppu_sprite0_scx1/2/5/6_b, sprite4_*, win*_a leftovers,
# win10_scx3_b). The remaining win*_b rows want a 2-dot flip lead on the
# lcd-enable line-1 window lines where the gambatte window/late_* rows
# pin the 1-dot stalled-window lead we model — a documented one-dot
# conflict (see baselines/gambatte.txt). lcdon_to_oam_int_l* and the
# remaining oam_int/int_oam rows are line-0/lcd-enable OAM-pulse chains
# not yet derived. dma_basic.gb can never report a verdict: it copies its
# DMA-window routine over the HRAM result block, so $FF82 reads back a
# code byte ($18, JR) — kept here as a permanent protocol-violation entry
# because it does write $FF82 within the deadline (see suite module docs).
gbmicrotest/dma_basic.gb [Dmg]
