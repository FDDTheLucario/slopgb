# wilbertpol known-failure baseline (consumed by tests/gbtr/wilbertpol.rs
# via harness::assert_against_baseline; one case key per line).
#
# Discovered 2026-06-11 by running the full 561-case matrix; every protocol
# failure below still exits via the 0xED protocol with non-Fibonacci
# registers (no timeouts). This fork carries ~40 LY/STAT-edge tests beyond
# the modern mooneye suite with 2016-era expectations, so many of these are
# expected to be legitimately hard; shrinking this list is the tracked
# progress, growing it is a regression.
#
# Failure clusters (re-baselined 2026-06: the STAT mode-2 pulse wave
# flipped intr_2_timing ×6 and stat_write_if-GS ×4 to pass):
#  - acceptance/gpu, CGB-only -C variants: the fork's CGB LY/LYC/STAT
#    register sequence expectations — ly_lyc family ×7 ROMs, ly00_mode1_2,
#    ly_new_frame, stat_write_if-C (the CGB-C LY/STAT timeline is not
#    modelled as a separate variant yet).
#  - acceptance/gpu, hblank_ly_scx_timing: the -C variant plus the
#    all-model _variant_nops build (the plain -GS variant and the _nops
#    build pass since the mode-0 event-grid re-derivation).
#  - acceptance/gpu, intr_2_mode0 family: the scx*/sprites*_nops variants
#    on their full 6-model matrices. These 2016-era chains measure the
#    visible mode-0 flip through wait_mode-poll-quantized multi-line
#    state, and no single flip assignment satisfies them together with
#    the modern grids: the scx4/scx8 builds rejected the old end-anchored
#    flip too, and the scx1/2/5/6 builds reject the re-derived pipe-end−2
#    flip that the gbmicrotest _if/halt grids, mooneye windows and the
#    gambatte event pair pin (documented swap: the re-derivation flipped
#    46 gbmicrotest + 117 gambatte + 2 mealybug + 2 age rows; siding with
#    these six builds instead would forfeit those — see the mode-0 grid
#    notes in ppu/render.rs `m0_flip_events`).
#  - madness (1): the 2016 build of mgb_oam_dma_halt_sprites renders no
#    HALT-glitch sprite under our MGB OAM-scan model (its 18 sprite pixels
#    come out as bare checkerboard), although the byte-different
#    mts-20240926 build passes the identical reference in tests/mooneye.rs.
