# oamdma/ residuals after the gambatte-core bus-conflict model landed
# (write-side conflicts, CGB WRAM redirect, CGB invalid >=E0 sources,
# CGB-C FEA0-FEFF extra RAM, FF46 mid-flight retarget), clustered by
# mechanism:
# * late_sp* + oamdma_late_halt_stat/late_speedchange_stat: sub-M-cycle
#   alignment of the DMA engine (start / halt-freeze / speed-switch)
#   against the PPU: hardware disconnects the PPU's OAM view at a precise
#   dot, raced against the dot-serial mode-2 scan of one sprite slot;
#   this emulator scans a line's OAM monolithically and steps the DMA at
#   M-cycle granularity (same mechanism as smallsuites strikethrough.gb).
# * srcFE00_* [Dmg]: DMA from $FE00 reads the $DE00 WRAM echo, whose
#   power-on content these ROMs never initialize; the expectations encode
#   the DMG power-up WRAM pattern ($DExx page = $FF, gambatte-core
#   mem_dumps.h setInitialDmgWram), slopgb zero-fills WRAM.
# * oamdmasrcC000_hdmasrc0000: the OAM-DMA x VRAM-DMA bus capture itself
#   is modelled (interconnect.rs oam_dma_bus_capture; the
#   hdma_transition_oamdma and oamdmasrc80_halt_*_read8000 siblings
#   pass), but this ROM races the steal against the *end* of the OAM DMA:
#   our block lands one M-cycle later on the OAM-DMA position grid, so
#   the last capture (block offset $0B) is cut off by the transfer's end
#   - the same M0-trigger/steal dot parity the dma/hdma_start_*_1 rows
#   pin (see the mode-3 swaps block below).
