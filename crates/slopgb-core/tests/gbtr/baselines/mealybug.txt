# mealybug-tearoom-tests known-failure baseline (one case key per line,
# consumed by gbtr/mealybug.rs via parse_baseline). Discovered by running
# mealybug_matrix against the pinned v7.0 collection; shrinking this list is
# progress, growing it is a regression (harness::assert_against_baseline).
#
# 26 of 55 rom x model legs fail. Passing: mbc/mbc3_rtc.gb (both),
# dma/hdma_during_halt-C (HBlank DMA defers across the core-clock gate
# and re-fires at the wake - the gambatte-shaped HDMA engine),
# m2_win_en_toggle [Cgb], m3_lcdc_bg_en_change/change2 [Cgb] (mode-0
# event-grid re-derivation), m3_lcdc_obj_en_change [Cgb] (LCDC.1 gates
# sprite pixels at the mix), and pixel-perfect both legs: m3_bgp_change,
# m3_scx_low_3_bits, m3_window_timing, m3_window_timing_wx_0,
# m3_lcdc_win_en_change_multiple, m3_wx_4_change_sprites (+ [Dmg]-only
# m3_wx_4/5/6_change), plus the fetch-grid wave's [Dmg] legs:
# m3_lcdc_tile_sel_change, m3_lcdc_tile_sel_win_change,
# m3_lcdc_bg_map_change, m3_lcdc_win_map_change, m3_scx_high_5_bits,
# m3_bgp_change_sprites, m3_obp0_change (fetch reads sample eff at the
# read dot; the BG fetcher free-runs through sprite stalls; the DMG blob
# pays 6 dots for every OBJ fetch with the mode-0 flip leading
# sprite-extended pipe ends by 3 — see ppu/render.rs).
# All ROMs reach LD B,B - no timeouts; every ppu failure is a frame
# mismatch against the bundled reference, every dma failure is the
# mooneye all-$42 fail-register signature.
#
# Floor classes (letters per the index in baselines/gambatte.txt):
# hdma_timing-C is class B (speed-switch/HDMA sub-M-cycle seam); the
# [Cgb] fetch/palette/scroll photo residue and the [Dmg] single-pixel
# legs below are class H sub-dot residue of the fetch grid (lift:
# per-stage CGB fetch structure / sub-dot write cells — the cluster
# comments name the hardware-captured gambatte rows that pin the
# opposite side, so don't force a single window one-sidedly).
