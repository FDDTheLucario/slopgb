# mealybug-tearoom-tests known-failure baseline (one case key per line,
# consumed by gbtr/mealybug.rs via parse_baseline). Discovered by running
# mealybug_matrix against the pinned v7.0 collection; shrinking this list is
# progress, growing it is a regression (harness::assert_against_baseline).
#
# 48 of 55 rom x model legs fail; mbc/mbc3_rtc.gb passes (both models),
# ppu/m2_win_en_toggle.gb passes on Cgb (its [Cgb] reference matches once
# the boot ROM's default DMG-compat palettes are installed; the [Dmg] leg
# still fails on the underlying mode-2 toggle behavior), and
# ppu/m3_bgp_change + ppu/m3_scx_low_3_bits pass pixel-perfect on both
# legs (mode-3 write strobe + line-0 OAM IRQ event semantics + SCX
# comparator hunt + boot-ROM logo tile data). All ROMs reach LD B,B — no
# timeouts; every ppu failure is a frame mismatch against the bundled
# reference, every dma failure is the mooneye all-$42 fail-register
# signature.

# dma/: HDMA timing — registers report the ROMs' fail code ($42 everywhere).
