# slopgb CGB boot ROM (original, open-source)

An **original** Game Boy Color boot ROM, written from public hardware
documentation (Pan Docs, the gbdev wiki, mooneye/gbmicrotest behaviour notes) â€”
not derived from any copyrighted boot ROM. It recreates the CGB power-on
experience frame-for-frame: hardware init, then the reference's animation

The SLOPGB wordmark is **original art** drawn in the chunky forward-italic style
of the classic Game Boy logo typeface (a typeface style is not protected; the
trademarked logo bitmap is never used or copied). It is generated by
`examples/gen_boot_logo.rs`; see **Logo art** below. The animation's exact
timing and colours were measured frame-by-frame from a legal boot-ROM backup
(the reference ROM's data and code are never copied â€” only its observable output
is matched). The 48-byte Nintendo logo a real cartridge carries at `$0104` is
**read from the cart** (mapped at `$0100-$01FF` during boot), never embedded
here.

## Logo art

`slopgb_logo.png` (96Ă—24, 1-bit) is the wordmark source. To regenerate it:

```sh
cargo run -p slopgb-core --example gen_boot_logo > /tmp/logo.pbm
magick /tmp/logo.pbm -type bilevel slopgb_logo.png   # PBM -> 1-bit PNG
make                                                  # rgbgfx packs logo.2bpp
```

The generator defines each glyph upright at half height, doubles it vertically
(the GB logo's signature chunk), and shears it into a forward italic.

## Build

```sh
make            # -> slopgb_cgb_boot.bin (2304 bytes, CGB-class)
```

Needs RGBDS (`rgbasm`/`rgblink`). Load it in slopgb with
`cargo run --release -- --boot boot/slopgb_cgb_boot.bin <game.gbc>` or via the
Options â†’ System bootrom path.

## Layout (the emulator's CGB-class mapping, `interconnect/boot_rom.rs`)

| Region | Contents |
|---|---|
| `$0000-$00FF` | entry + main boot routine; jumps past the header gap to `$0200` |
| `$0100-$01FF` | **cart header** shows through here (logo + title) â€” not boot ROM |
| `$0200-$08FF` | continuation: logo tiles, palette tables, chime, hand-off |

`FF50` bit 0 written 1 unmaps the boot ROM and hands control to the cart at
`$0100`.

## CGB compatibility palettes (P5) â€” provenance

The per-game palette assignment is **factual hardware-interop data** (the same
category as the core's existing `CGB_COMPAT_*` palettes), reproduced clean-room:

1. The selection **algorithm** (checksum = ÎŁ title bytes $0134-$0143 â†’ table search
   â†’ 4th-letter $0137 collision tiebreak â†’ palette assignment) is from public
   documentation (Pan Docs "Power-Up Sequence", TCRF) and confirmed against the
   reference ROM's disassembly.
2. The palette **values** are recovered by *black-box observation*: synthetic
   carts are booted through the reference `cgb_boot.bin` and the palette it
   installs is read back (`crates/slopgb-core/examples/cgb_palette_extract.rs`).
   No reference ROM code or data layout is copied â€” only the titleâ†’palette

3. `cgb_palette_extract ... verify <our.bin>` boots **every** checksum + collision

`cgb_palettes.inc` is generated; regenerate with
